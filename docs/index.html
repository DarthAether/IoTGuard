// Key Features

Capabilities

🧠

LLM Risk Analysis

Google Gemini-powered command analysis that understands device context, command semantics, and potential security implications beyond simple pattern matching.

📜

Regex Security Rules

Configurable rule engine with regex pattern matching for fast, deterministic command filtering before LLM analysis.

📡

MQTT Device Bridge

Native MQTT integration for bidirectional device communication. Publish verified commands and subscribe to device telemetry.

🔒

JWT + RBAC

Role-based access control with JWT authentication. Fine-grained permissions for device management, rule configuration, and command execution.

⚡

Redis Caching

Redis-backed caching layer for LLM responses and rule evaluations, reducing latency on repeated command patterns.

📊

Prometheus + Grafana

Full observability stack with custom metrics for command throughput, risk score distribution, block rates, and MQTT health.

// Quick Start

Get Running

terminal 
# Clone the repository
git clone https://github.com/DarthAether/IoTGuard.git
cd IoTGuard

# Copy environment config
cp .env.example .env
# Add your Gemini API key to .env

# Start all services
docker-compose up -d

# Access services
# API:        http://localhost:8000/docs
# Grafana:    http://localhost:3000
# Prometheus: http://localhost:9090

# Run tests
docker-compose exec api pytest -v
// API Overview

Key Endpoints

Method Endpoint Description
POST /api/v1/analyze Analyze IoT command for security risks
GET /api/v1/devices List registered IoT devices
POST /api/v1/devices Register a new IoT device
GET /api/v1/rules List security rules
POST /api/v1/rules Create a new security rule
POST /api/v1/mqtt/publish Publish command to MQTT device
GET /metrics Prometheus metrics endpoint
